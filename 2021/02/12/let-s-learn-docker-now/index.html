Docker基本概念


镜像image：镜像就好比是一个模板，通过镜像run，创建容器服务，镜像可以创建多个容器
容器container：利用容器独立运行一个或一组应用。启动、停止、删除。可以把容器理解为一个简单的Linux系统
仓库repository：存放镜像的地方

#安装docker
sudo yum install docker-ce docker-ce-cli containerd.io
#启动docker 
sudo systemctl start docker   
#测试是否安装成功

#测试hello world

Docker原理

Docker运行过程

Docker组件原理

Docker是一个client-server结构的系统，Docker的守护进程运行在主机上，通过websocket从客户端进行访问。
Docker-server接收Docker-client的指令

为什么Docker比VM快

1、Docker有比虚拟机更少的抽象层
2、Docker使用的是宿主机的内核，虚拟机需要的Guest OS。新建容器的时候，Docker不需要重新加载一个操作系统

容器命令

新建容器并启动

# docker run [可选参数] image
# 参数说明
--name="name1"  容器启动后的名字，用于区分
-d 后台运行，类似 nohup
-it 使用交互方式运行，进入容器查看容器
-p 8080:8080 指定容器端口。ip:主机端口:容器端口(常用)
-P 随机指定端口

查看正在运行的容器

docker ps #正在运行的容器
CONTAINER ID   IMAGE     COMMAND   CREATED   STATUS    PORTS     NAMES

docker ps -a #查看曾经运行的所有容器
CONTAINER ID   IMAGE     COMMAND       CREATED         STATUS                     PORTS     NAMES
8fc216825a94   centos    "/bin/bash"   5 minutes ago   Exited (0) 2 minutes ago             flamboyant_ptolemy

退出容器

ctrl+p+q (退出但是不停止)或者 exit (退出并停止)

删除容器

docker rm 容器id 							#没法删除正在运行的容器
docker rm -f 容器id
$ docker ps -a -q | xargs docker rm -f 		#删除所有容器的骚操作
715959ece1f0
8fc216825a94

启动和停止容器

docker start 容器id 	#启动容器。对比run，run是有创建的过程，start是已经存在的容器s
docker restart 容器id
docker stop  容器id	#停止容器
docker kill 容器id 	#强制停止容器，当stop不好使的时候

问题

我在使用docker后台启动的时候，发现他并没有在运行，自己退出了。
原因：
这是因为docker在启动后，需要有前台服务，当发现自己没有提供服务，就自杀了。发现自己没用，就自杀了

进入当前正在运行的容器

方式一

#docker exec -it 容器id bashshell
docker exec -it 90ac614554b0 /bin/bash
[root@90ac614554b0 /]# ls
bin  dev  etc  home  lib  lib64  lost+found  media  mnt  opt  proc  root  run  sbin  srv  sys  tmp  usr  var
[root@90ac614554b0 /]# ps 
  PID TTY          TIME CMD
   24 pts/1    00:00:00 bash
   40 pts/1    00:00:00 ps

方式二

#docker attach 容器id
docker attach 90ac614554b0
[root@90ac614554b0 /]#

二者的区别
docker exec 是以新的终端形式进入容器
docker attach 是直接进入正在运行的终端
